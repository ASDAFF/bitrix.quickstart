В версии 1.3.2 добавлен автоматический режим заполнения полей почтового шаблона, теперь там достаточно вписать один макрос #WORK_AREA#

На сайте обновлена документация для версии 1.3.2

В следующих обновлениях добавлю возможность задавать свои названия полей для почтового шаблона в автоматическом режиме, сейчас там заполняются пока те, что выводятся на сайте пользователю.

Скриншоты Обновления Поддержка Установка
Скриншоты
Обновления
1.3.8 - полностью переработаны заголовки письма, у меня письма больше не попадают в спам, конкретно доработаны под требования хостинга http://beget.ru?id=81076 в котором при особых условиях подменяется e-mail отправителя на e-mail хостинга в целях борьбы с массовым спамом; - исправлена ошибка, когда поле e-mail конструктора форм не участвовало в подмене адреса отправителя при включенной опции "Подставлять в поле "От" e-mail посетителя"; - исправлена ошибка, когда при макросе #FILES# все тело письма ниже обрезалось. p.s.: На некоторых хостингах опция "Подставлять в поле "От" e-mail посетителя" может не работать или отправлять письма в спам, т.к. происходит подмена e-mail реального отправителя письма.
1.3.7 - исправлена инициализация jQuery-валидатора;
- исправлена ошибка в модуле PHP-антиспам.
1.3.6 - переработан CSS-антиспам во всех шаблонах;
- добавлен PHP-антиспам с указанием разрешенного количества совпадений для текстовых полей, подробно читайте в подсказке к опции в настройках компонента, по умолчанию отключен, включается в настройках компонента;
- настройки CAPTCHA и антиспама вынесены в отдельный раздел;
- несколько мелких исправлений по коду и описаниям.
1.3.5 - улучшен антиспам во всех шаблонах. Пробуйте отключать стандартную капчу, он практически бесполезна.
- в component.php все $_POST заменил на $_REQUEST, чтобы работали и $_GET и $_POST ajax-запросы.
1.3.4 - исправлена ошибка, когда выводилась информация об ошибке, при включенной копии сообщения посетителю, а поле e-mail не обязательное;
- все возможные заполненные в профиле пользователя поля подставляются в форме автоматически, кроме Администратора.
1.3.3 - в почтовый шаблон добавлен макрос #HTTP_USER_AGENT# - эта строка содержит обозначение браузера, которым пользователь запросил данную страницу, например: Mozilla/4.5 [en] (X11; U; Linux 2.2.9 i586).
- для полей конструктора типа text, date, textarea добавлена возможность задать значения по умолчанию атрибутом @value=, которые будут выводиться в форме и отправляться на почту, как заполненные, например: Текстовое поле@input@text@required@value=Значение по умолчанию
1.3.2 - добавлен автоматический режим заполнения почтового шаблона в виде одного макроса #WORK_AREA# в почтовом шаблоне, больше не нужно писать в нем вручную названия полей и макросы, также сделана совместимость, присутствующие в шаблоне макросы будут также подменяться;
- добавлена возможность переопределять e-mail администратора из настроек компонента в поле Кому в почтовом шаблоне административной панели, все письма будут отправляться на вписанный в поле Кому e-mail вместо макроса #EMAIL_TO#; - добавлены системные макросы: IP отправителя:#IP#, Дата и время:#DATETIME#, системные макросы пишутся в почтовом шаблоне вручную;
- добавлена опция "Очищать папку с файлами" после отправки сообщения. В этом случае отправка ссылок на файлы невозможна, т.к. файлы будут удалены на сервере, в этом случае будет автоматически работать опция "Отправлять как вложения";
- добавлена транслитерация кириллических(русских) имен файлов, т.к. некоторые почтовые программы, например, Mozilla Thunderbird, не распознают пробелы и имя файла получается разорвано, соответственно перейдя по ссылке файл не будет найден, а если еще на хостинге не настроена поддержка кириллических имен файлов, то имя файла при загрузке становится кракозябрами;
- исправлена ошибка, когда не подменялось не введенное пользователем поле e-mail, на e-mail из поля от кого, при включенной опции: Подставлять в поле "От" e-mail посетителя;
- исправлена ошибка, когда вместо пустых интервалов дат добавлялась одна запятая;
- исправлена ошибка с вложениями файлов, когда макрос #FILES# в почтовом шаблоне был без переноса строки, вложения отображались кракозябрами в письме;
- исправлена ошибка, когда не добавлялись макросы почтового шаблона в Тип почтового события FEEDBACK_FORM;
- при обязательной загрузке файлов выводилось сообщение Введите "Файл", заменено на Введите "Все файлы";
- переписаны некоторые ошибки с рекомендациями по их устранению;
- в случае многосайтовой конфигурации макрос #SITE_NAME# - Название сайта будет браться из секции Параметры->Название сайта на странице редактирования сайта, иначе берется из Названия сайта в настройках Главного модуля;
- теперь в почтовых шаблонах модуля при установке по умолчанию прописан макрос #WORK_AREA# для автоматического режима работы;
1.3.1 1. Добавлены во все системные шаблоны кроссбраузерные placeholder'ы ко всем полям и соответствующая опция в настройках компонента для их включения;
2. Исправлена ошибка обработки поля типа E-mail в конструкторе форм, синтаксис: E-mail@input@email@required
1.3.0 1) Исправлена ошибка, когда при копировании шаблона не отображались иконки jQuery валидатора формы.
2) Добавлен плагин стилизации флажков/переключателей jQuery Form Styler.
